Product Requirements Document (PRD): Unified Agentic OS & Hermes Workspace
1. Product Vision & Objectives
The Agentic OS Dashboard is a self-hosted, single-user "Mission Control" designed to orchestrate a fleet of autonomous AI agents (Claude Code, Hermes, and local models). It bridges the gap between raw LLM reasoning and professional infrastructure management by integrating persistent memory (Mem0), structured knowledge (Obsidian), and real-time system monitoring into a single unified workspace.
2. Technical Architecture
• Deployment: Docker-first architecture; optional Electron-based desktop wrapper for easier local installation and configuration.
• Networking: Native PWA support for mobile; built-in Tailscale integration for secure remote access via personal Tailnets.
• Security: Auth middleware on every route; CSP and path-traversal guards; backend stored in an encrypted SQLite database.
• Zero-Fork Compatibility: Deep integration with vanilla hermes-agent via standard API endpoints, ensuring compatibility with upstream updates.
3. Functional Requirements
3.1 Agent Swarm & Mission Control
• Conductor (Mission Dispatch): A mission-control surface to decompose complex requests into smaller tasks, assign them to specialized workers, and watch live output.
• Swarm Mode: Persistent tmux-backed worker processes that maintain context across long-running tasks, rotate safely, and report proof-bearing checkpoints.
• Specialized Agent Roles: Dedicated lanes for specific functions: Researcher (codebase analysis), Builder (execution), Validator (reviewing diffs), and Scribe (documentation).
• Multi-Agent Status: Live monitoring of local and remote agents; visual indicators for "Thinking," "Waiting," and "Idle" status; @mention routing for triggering specific agents in group chats.
